🌳 夢想森林：維護與更新指南

當你們將網頁部署到 GitHub 並分享連結後，後續的更新主要分為以下兩種情況：

1. 內容更新 (願望、檔案、紀念日)

— 100% 自動同步，無需重新操作 —

由於網頁後台連接了 Firebase 雲端資料庫：

即時性：糖糖在手機上新增一個願望，果果的螢幕會「立刻」跳出新的許願瓶，完全不需要重新整理網頁。

持久性：你們填寫的身高體重、點選的灌溉紀錄，都會永久儲存在雲端。即使你們關閉網頁、換一支手機登入，資料依然會在那裡。

結論：只要是網頁內的「互動內容」，你們隨時隨地更新，對方看到的都會是最新的森林。

2. 功能與外觀更新 (程式碼修改)

— 需透過 GitHub 重新發布 —

如果你想要修改網頁的「程式碼」（例如：想換背景顏色、想增加新的個人資料欄位、或是想調整小樹長大的速度）：

修改檔案：在你的電腦上修改 index.html 的代碼。

上傳 GitHub：將新的代碼上傳（Push）到 GitHub 的倉庫（Repository）。

自動部署：GitHub Pages 會在大約 1 分鐘內自動偵測到新版本並完成部署。

對方如何看到：對方只需要重新整理網頁（或是關閉分頁再重新開啟），就會看到全新的功能或設計了。

提示：你們分享的「網址」永遠不會變，所以不需要重新傳連結給對方！

💡 維護小秘訣

清快取：有時候因為瀏覽器的緩存（Cache），功能更新後對方可能沒看到變化，這時請對方「強制重新整理」即可。

備份金鑰：請妥善保管你的 firebaseConfig，那是你們森林唯一的鑰匙。

測試環境：建議在把代碼上傳 GitHub 前，先在本地端打開檔案測看看有沒有 Bug，確認沒問題再發布給另一半看，保持驚喜感！

祝糖糖與果果的森林永遠生機盎然！✨
